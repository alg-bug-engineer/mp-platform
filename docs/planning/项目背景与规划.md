# 项目背景与规划（2026-02-18）

## 1. 项目背景

`we-mp-rss` 最初是公众号订阅抓取与 RSS 输出工具。随着内容行业从“采集分发”转向“AI 规模化创作”，项目目标升级为：

- 面向自媒体博主的一站式创作平台
- 打通“选题 -> 生成 -> 配图 -> 发布 -> 复盘”的闭环
- 支持不同规模用户的商业化分层服务

## 2. 产品目标

### 2.1 北极星目标

- 让单个创作者在 30 分钟内完成一篇可发布图文
- 让团队用户具备批量产出与稳定发布能力
- 形成可持续商业化模型（免费试用 + 付费转化 + 高级扩展）

### 2.2 关键指标（建议）

- DAU/WAU 创作者数
- 人均周创作篇数
- AI 任务成功率、草稿投递成功率
- 免费 -> 付费转化率、续费率、ARPU

## 3. 当前现状（已落地）

- 账号体系：手机号注册 + 登录，支持管理员账号
- 数据隔离：按用户隔离公众号、文章、标签、消息任务、AI 配置
- AI 中台：
  - 一键分析 / 创作 / 仿写
  - 平台风格模板与本地规则
  - 即梦图片生成（可降级提示词）
  - 本地草稿箱
  - 公众号草稿箱投递（支持失败入队重试）
- 商业化：
  - 免费 / 付费 / 高级套餐模型
  - AI 与图片配额控制
  - 草稿投递权限分层
- 管理能力：
  - 管理员套餐管理页（调档、配额调整、重置消耗）

## 4. 功能地图

### 4.1 创作者端

- 内容源管理：订阅公众号、文章检索
- AI 创作：分析、改写、跨平台风格生成
- 媒体增强：即梦封面 / Logo / 素材图
- 发布管理：本地草稿、公众号草稿投递、投递队列重试

### 4.2 管理端

- 用户列表与套餐管理
- 配额策略调整
- 运营策略配置（模型、规则、任务）

## 5. 开发计划（滚动）

### 阶段 A（已完成）

- 创作中台 UI 重构
- 套餐分层与配额体系
- 草稿箱投递与失败队列

### 阶段 B（已完成）

- 公众号授权按用户维度隔离（`wechat_auths`）
- 草稿投递失败入队 + 批量处理 + 单条重试
- 支付订阅闭环（订单、支付确认、订阅生效、到期扫描降级）
- 套餐管理后台增强（调档、配额重置、边界校验）

### 阶段 C（规划）

- 团队协作能力（子账号、审批流、角色权限）
- 财务能力（真实支付网关、发票、对账）
- 内容增长闭环（A/B 标题测试、发布后数据回收）

## 6. 风险与约束

- 微信公众平台接口策略可能变化，需保持降级与重试机制
- AI 成本与质量波动，需做模型路由与成本监控
- 商业化推进依赖支付系统与法务合规（隐私、版权、服务协议）
